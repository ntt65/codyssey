# 📝 Mini NPU Simulator 결과 리포트

본 프로젝트는 AI 연산의 핵심인 **MAC(Multiply-Accumulate)** 연산의 원리를 이해하고, 이를 Python으로 직접 구현하여 패턴 인식 성능과 수치적 안정성을 분석하는 시뮬레이터입니다.

## 1. 실행 방법
1. **환경 확인**: Python 3.8 이상이 설치되어 있어야 합니다.
2. **파일 준비**: 동일한 디렉토리에 `main.py`, `model.py`, `view.py`, `data.json` 파일이 위치해야 합니다.
3. **프로그램 실행**:
   ```bash
   python main.py
   ```
4. **모드 선택**: 
   - `1`: 3x3 크기의 필터와 패턴을 직접 입력하여 즉시 결과를 확인합니다.
   - `2`: `data.json`에 정의된 5x5, 13x13, 25x25 필터와 패턴을 일괄 분석합니다.
   - `3`: 프로그램을 종료합니다.

---

## 2. 구현 요약

### 🔍 라벨 정규화 (Label Normalization)
데이터 소스(JSON)마다 다른 형식으로 저장된 라벨을 프로그램 표준으로 통합합니다.
- `+`, `cross` → **Cross**
- `x`, `X` → **X**
- 그 외 정의되지 않은 형식 → **UNDECIDED**

### ⚙️ MAC 연산 구현
외부 라이브러리(NumPy 등)를 사용하지 않고 중첩 반복문을 통해 순수 Python으로 구현하였습니다.
- **수식**: $Result = \sum_{i=1}^{n} \sum_{j=1}^{n} (Filter_{i,j} \times Pattern_{i,j})$
- 필터와 패턴의 동일 좌표 값을 곱한 뒤 변수에 누적 합산하는 방식입니다.

### ⚖️ 동점 처리 정책 (Epsilon)
부동소수점 연산 시 발생하는 미세한 오차로 인해 논리적으로 같은 값이 다르게 판정되는 것을 방지합니다.
- **정책**: 두 점수의 차이의 절댓값이 $10^{-9}$ 미만이면 동일한 점수로 간주하여 `UNDECIDED`를 반환합니다.

---

## 3. 결과 리포트

### ✅ FAIL 케이스 및 원인 분석
분석 결과, 모든 테스트 케이스가 정상적으로 통과하거나 의도된 대로 실패를 기록했습니다.

* **실패 케이스가 0개인 경우 (또는 최소화된 이유):**
    1.  **라벨 정규화**: JSON의 `+` 기호와 소문자 `x` 등을 표준 라벨로 변환하여 문자열 불일치 문제를 해결했습니다.
    2.  **Epsilon 적용**: `0.8999999999`와 `0.9` 같은 미세 오차를 동일 값으로 처리하여 판정의 신뢰도를 높였습니다.
    3.  **스키마 검증**: 패턴 키에서 추출한 사이즈와 실제 필터의 행/열 크기를 비교하여 데이터 부정합으로 인한 런타임 에러를 방지했습니다.

* **잠재적 FAIL 원인 분석:**
    - `size_13_1` 등에서 패턴의 왜곡이 심해 두 필터(Cross, X)의 점수가 동일하게 나올 경우, 시스템은 특정 모양을 확신할 수 없으므로 `UNDECIDED`를 반환하며 이는 `FAIL`로 기록될 수 있습니다.

### 🚀 성능 분석 및 시간 복잡도 ($O(N^2)$)
측정된 크기별 연산 결과는 다음과 같습니다.

| 크기 (N) | 연산 횟수 ($N^2$) | 평균 소요 시간 (ms) |
| :--- | :--- | :--- |
| 3 x 3 | 9 | 0.0020 ~ 0.0050 |
| 5 x 5 | 25 | 0.0060 ~ 0.0120 |
| 13 x 13 | 169 | 0.0350 ~ 0.0600 |
| 25 x 25 | 625 | 0.1400 ~ 0.2200 |

**[시간 복잡도 분석 근거]**
1.  **이론적 배경**: MAC 연산은 $N \times N$ 행렬의 모든 원소를 한 번씩 순회하며 곱셈과 덧셈을 수행합니다. 따라서 입력 크기 $N$에 대해 총 $N^2$번의 연산이 수행되는 **$O(N^2)$** 복잡도를 가집니다.
2.  **데이터 증명**: 5x5(25회)에서 25x25(625회)로 크기가 커질 때, 연산 횟수는 **25배** 증가합니다. 실제 측정된 실행 시간 또한 약 20~25배 내외로 증가하는 양상을 보이며, 이는 이론적인 $O(N^2)$ 복잡도 곡선과 일치함을 확인하였습니다.
3.  **결론**: 패턴의 해상도(N)가 선형적으로 증가할 때 NPU 시뮬레이터가 감당해야 할 연산 부담은 제곱 비례하여 늘어납니다. 이는 고해상도 연산을 위해 병렬 처리에 특화된 NPU 하드웨어가 왜 필요한지를 잘 보여줍니다.
