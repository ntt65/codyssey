# 대화형 가계부 콘솔 프로그램 통합 설계 계획서 (Revised mission_plan.md)

본 계획서는 프로그램 구동 방식부터 모든 셸 명령어(14종)의 구체적인 제어 흐름, 입출력 정의, 예외 처리 설계까지 하나도 빠짐없이 포함하는 상세 사양서입니다. 사용자와 합의 후 개발에 착수하는 최종 계약서 역할을 수행합니다.

---

## 1. 전역 및 셸 구동 설계

### 1.1 실행 및 공통 매개변수
* **실행 명령어**: `python3 -m budget_app [--data-dir 경로]`
* **데이터 폴더 구성**:
  * 기본 경로: `./data`
  * 파일 리스트: `transactions.jsonl`, `categories.jsonl`, `budgets.jsonl`, `recurring.jsonl`
* **자동 초기화**: 실행 시 해당 디렉터리가 없으면 자동 생성하며, `categories.jsonl` 파일이 비어있는 경우 기본 카테고리 9종(`food`, `transport`, `rent`, `shopping`, `health`, `education`, `salary`, `allowance`, `other`)을 즉시 기입해 자동 생성합니다.

### 1.2 셸 주 루프 (Main Shell Loop) 구조
* 프로그램 진입 시 웰컴 배너와 도움말 안내를 출력한 뒤, `while True` 무한 루프에서 사용자 입력을 받습니다.
* 프롬프트 표기: `budget_app> `
* **예외 격리**: 명령어 실행 도중 에러가 나더라도 루프가 깨지거나 비정상 종료되지 않고, 에러 출력 후 다시 `budget_app> ` 프롬프트 대기 상태로 순환합니다.
* **명령 파싱 규칙**: 입력 한 줄을 공백 기준으로 쪼개어 첫 단어를 명령어, 이후 단어를 매개변수 인자로 다룹니다.

---

## 2. 14종 명령어별 상세 대화식 입력 및 처리 설계

### 2.1 [help] - 도움말 및 명령어 안내
* **진입 조건**: `help` 입력
* **동작 및 출력**:
  * 셸 내부에서 사용 가능한 모든 명령어와 아규먼트 구조를 표 형식으로 출력합니다.
  ```text
  [ 가용 명령어 가이드 ]
  --------------------------------------------------------------------------------
  명령어       | 인자 형태       | 한글 기능 설명
  --------------------------------------------------------------------------------
  help         | 없음            | 도움말 및 명령어 안내를 출력합니다.
  add          | 없음            | 새로운 수입/지출 내역을 입력 루프로 등록합니다.
  list         | [limit]         | 최신 가계부 내역을 정렬 표 형식으로 출력합니다.
  search       | 없음            | 조건별(기간, 카테고리 등) 대화형 검색을 수행합니다.
  summary      | [YYYY-MM]       | 지정한 월의 수지 통계 및 예산 소모를 분석합니다.
  budget       | 없음            | 특정 월의 소비 한도 예산을 입력 설정합니다.
  category     | 없음            | 카테고리 조회/추가/삭제 서브 메뉴로 진입합니다.
  update       | [--id ID]       | 특정 ID 거래 건을 대화식으로 덮어써 수정합니다.
  delete       | [--id ID]       | 특정 ID 거래 건을 확인 절차를 거쳐 삭제합니다.
  import       | [--from 파일]   | 외부 CSV 파일의 대량 데이터를 로드 및 등록합니다.
  export       | [--out 파일]    | 필터링된 범위의 가계부 데이터를 CSV로 추출합니다.
  backup       | 없음            | 데이터베이스 전체를 zip 파일로 일괄 백업합니다.
  recurring    | 없음            | 반복 거래(고정비) 템플릿 관리 및 생성으로 진입합니다.
  exit / quit  | 없음            | 프로그램을 안전하게 세이브하고 종료합니다.
  --------------------------------------------------------------------------------
  ```

### 2.2 [add] - 신규 거래 추가
* **진입 조건**: `add` 입력
* **동작 및 흐름**:
  1. `날짜(YYYY-MM-DD) [기본값: 오늘]:` 입력 대기. 
     * 엔터 시 오늘 날짜 자동 할당. 날짜 형식 오류(예: 2024-02-30) 시 에러 및 힌트 출력 후 다시 질문.
  2. `타입(income/expense):` 입력 대기.
     * `income` 또는 `expense` 외 오입력 시 경고 후 재질문.
  3. `카테고리:` 입력 대기.
     * 입력값이 기존 카테고리 목록에 없으면:
       `[오류] 존재하지 않는 카테고리입니다.`
       `[힌트] 등록된 카테고리: food, transport...`
       `'입력값' 카테고리를 새로 추가하고 진행하시겠습니까? (y/n):`
       * `y` 입력 시 자동으로 카테고리 추가 처리 후 계속 진행. `n` 입력 시 재입력 대기.
  4. `금액(양수 정수):` 입력 대기.
     * 문자열이거나 0 이하일 경우 에러 후 재질문.
  5. `메모(선택, 없으면 엔터):` 입력 대기.
  6. `태그(쉼표 구분, 없으면 엔터):` 입력 대기. (예: `외식,한식` -> `['외식', '한식']` 리스트 파싱)
  7. **최종 저장**: `TX-XXXXXX` 형태의 고유 일련번호 ID를 채번하여 파일 끝에 추가 저장하고 성공 메시지 출력.
  ```text
  [저장 완료] id=TX-000001
  ```

### 2.3 [list] - 목록 조회
* **진입 조건**: `list` 또는 `list <limit>` 입력 (예: `list 5`)
* **동작 및 흐름**:
  1. 인자로 넘어온 limit을 분석하여 양의 정수인지 검증합니다. 생략 시 기본값 `20`을 할당합니다.
  2. 전체 파일을 메모리에 적재하지 않고, 파일 끝에서부터 스캔하거나 실시간 스트리밍 제너레이터 동작 중 `limit` 크기의 삽입 정렬 큐를 유지해 메모리를 O(limit)로 고정합니다.
  3. 2바이트 한글 문자와 1바이트 영문/숫자 폭을 계산하여 컬럼 너비를 유동적으로 계산하는 정렬 테이블 형식으로 출력합니다.
  ```text
  id        | date       | type    | category | amount | memo  | tags
  TX-000002 | 2024-01-16 | income  | salary   | 300000 | 월급  | monthly
  TX-000001 | 2024-01-15 | expense | food     | 15000  | 점심  | 외식,식사
  ```

### 2.4 [search] - 복합 필터 검색
* **진입 조건**: `search` 입력
* **동작 및 흐름**:
  1. 사용자에게 아래의 필터링 질문을 순차적으로 수행합니다. (입력 없이 엔터를 치면 해당 조건 필터 해제)
     * `검색 시작일 (YYYY-MM-DD, 미지정은 엔터):`
     * `검색 종료일 (YYYY-MM-DD, 미지정은 엔터):`
     * `타입 필터 (income/expense, 미지정은 엔터):`
     * `카테고리 필터 (미지정은 엔터):`
     * `메모 검색어 (미지정은 엔터):` (대소문자 무관 부분 매칭)
     * `태그 필터 (미지정은 엔터):` (태그 리스트 내 완전 매칭)
  2. 스트리밍으로 한 레코드씩 읽으며 상기 필터 조건에 부합하는지 즉석 검증하여 매칭되는 레코드만 취합(최대 50개)합니다.
  3. 필터링된 결과를 `list`와 동일한 폭 맞춤 테이블로 렌더링합니다.

### 2.5 [summary] - 월간 재정 분석 및 예산 비교
* **진입 조건**: `summary` 또는 `summary YYYY-MM` 입력 (예: `summary 2024-01`)
* **동작 및 흐름**:
  1. 인자가 생략된 경우 `요약할 대상 월 (YYYY-MM) [기본값: 이번달]:` 프롬프트를 띄워 월을 입력받습니다.
  2. 데이터베이스를 스트리밍하여 해당 월에 해당하는 수입 총합, 지출 총합, 카테고리별 지출 누적 합계를 구합니다.
  3. 지출 카테고리 누적액을 내림차순 정렬하여 상위 3개(TOP 3) 항목을 백분율과 함께 가공합니다.
  4. 예산 설정 내역(`budgets.jsonl`)에서 해당 월의 한도를 조회합니다.
     * 예산이 존재할 경우: 지출 비율(%)을 연산하여 출력하고, 초과 시 경고 기호(`⚠️ [경고]`)와 초과 액수를 붉은 텍스트 및 경고음 등으로 출력합니다.
     * 예산이 없는 경우: 예산 정보를 생략하고 수지만 보여줍니다.
  5. 해당 월의 데이터가 전혀 없다면 `데이터 없음`을 명확히 출력하고 복귀합니다.

### 2.6 [budget] - 예산 한도 설정
* **진입 조건**: `budget` 입력
* **동작 및 흐름**:
  1. `예산을 책정할 대상 월 (YYYY-MM):` 입력 대기 및 포맷 유효성 검증.
  2. `설정할 예산 금액 (0 이상의 정수):` 입력 대기 및 유효성 검증.
  3. 기존 해당 월 예산 기록이 있다면 덮어쓰고, 없으면 신규 저장하여 atomic한 쓰기 기법으로 세이브를 마친 뒤 확인 통지를 출력합니다.

### 2.7 [category] - 카테고리 관리 메뉴
* **진입 조건**: `category` 입력
* **동작 및 흐름**:
  1. 사용자에게 작업 선택 하위 메뉴를 노출합니다:
     ```text
     [ 카테고리 설정 관리 ]
     1. 등록된 카테고리 목록 조회
     2. 신규 카테고리 추가
     3. 기존 카테고리 삭제
     선택 (1/2/3/엔터(취소)): 
     ```
  2. 분기 흐름:
     * **1번 (조회)**: 기등록 카테고리를 한 줄에 하나씩 `- 카테고리명` 포맷으로 열거합니다.
     * **2번 (추가)**: `추가할 카테고리명:` 대기 및 중복 검증 수행 후 추가 기록.
     * **3번 (삭제)**: `삭제할 카테고리명:` 대기.
       * **중요 보호**: 전체 트랜잭션을 스트리밍하여 해당 카테고리가 가계부에 등록되어 작동 중인지 실시간 검사합니다. 사용 중인 카테고리일 경우 삭제를 강제 기각하고 에러 원인을 안내합니다. 사용되고 있지 않으면 안전하게 unregister하고 atomic하게 디스크에 저장합니다.

### 2.8 [update] - 거래 내역 수정
* **진입 조건**: `update` 또는 `update --id <ID>` 입력
* **동작 및 흐름**:
  1. 인자로 ID를 지정하지 않았을 경우 `- 수정할 거래 ID:` 입력을 받습니다.
  2. 전체 파일을 스트리밍하여 일치하는 고유 ID가 존재하는지 체크하고, 존재하지 않으면 에러 메시지와 함께 셸 루프로 바로 복귀합니다.
  3. 수정 대상 거래를 읽어온 뒤 각 필드에 대해 차례로 질문합니다:
     * `날짜(YYYY-MM-DD) [기존값]:`
     * `타입(income/expense) [기존값]:`
     * `카테고리 [기존값]:` (카테고리 미등록 시 add와 동일한 신규 등록 유도 분기 개입)
     * `금액 [기존값]:`
     * `메모 [기존값]:`
     * `태그 [기존값]:`
  4. 질문 중 사용자가 값을 기입하지 않고 바로 엔터를 치면 **기존 값으로 기본 유지**시킵니다. 값을 기입하면 해당 값에 대한 유효성 검증을 거쳐 값을 정정합니다.
  5. 임시 파일을 생성하여 변경된 레코드를 덮어쓰고 `os.replace`를 통해 안전하게 치환합니다.

### 2.9 [delete] - 거래 삭제
* **진입 조건**: `delete` 또는 `delete --id <ID>` 입력
* **동작 및 흐름**:
  1. 인자로 ID를 지정하지 않았을 경우 `- 삭제할 거래 ID:` 입력을 받습니다.
  2. 대상 ID의 존재 유무를 확인하고, 없으면 에러 출력 후 복귀합니다.
  3. ID가 유효하면 `정말로 거래 ID 'TX-XXXXXX'를 영구 삭제하시겠습니까? (y/n):` 최종 컨펌 프롬프트를 띄웁니다.
  4. `y` 응답 시 임시 파일 쓰기와 atomic rename 기법으로 해당 줄을 배제한 채 저장소를 갱신한 뒤 성공 메시지를 출력합니다.

### 2.10 [import] - CSV 가져오기
* **진입 조건**: `import` 또는 `import --from <경로>` 입력
* **동작 및 흐름**:
  1. 인자로 경로를 지정하지 않았을 경우 `- 가져올 CSV 파일 경로 (예: data.csv):` 입력을 받습니다.
  2. 파일 유무 검증 후, `csv.DictReader`로 데이터를 순차 파싱합니다.
  3. 헤더 누락 검사(`date`, `type`, `category`, `amount` 필수)를 거쳐 하나라도 없으면 에러 처리 후 기각합니다.
  4. 각 행의 데이터를 유효성 분석하여 적격 판정 시 순차 고유 ID를 부여해 즉시 transactions 파일에 덧붙이고, 결격 사유가 발견되면 스킵 횟수를 증가시켜 최종 건수를 보고합니다:
     ```text
     [완료] CSV 처리 완료 - 가져옴: N건, 건너뜀(오류): M건
     ```

### 2.11 [export] - CSV 내보내기
* **진입 조건**: `export` 또는 `export --out <경로>` 입력
* **동작 및 흐름**:
  1. 인자로 경로를 지정하지 않았을 경우 `- 내보낼 CSV 파일 경로 (예: backup.csv):` 입력을 받습니다.
  2. 필터 범위 설정을 위해 대화식 분기를 띄웁니다:
     ```text
     - 필터 방식 선택 (1: 특정 월, 2: 특정 기간 범위, 엔터: 전체 내보내기): 
     ```
  3. 1번 선택 시 월(`YYYY-MM`)을 묻고, 2번 선택 시 시작일(`YYYY-MM-DD`) 및 종료일(`YYYY-MM-DD`)을 물어 필터링 컨텍스트를 완성합니다.
  4. 데이터베이스 스트리밍을 돌려 조건에 부합하는 레코드를 타겟 파일에 UTF-8 인코딩 및 헤더를 포함한 CSV 데이터로 스트리밍 기록하고 성공 메시지를 띄웁니다.

### 2.12 [backup] - 압축 백업본 생성
* **진입 조건**: `backup` 입력
* **동작 및 흐름**:
  1. 별도 프롬프트 없이 즉각 백업 기능을 구동합니다.
  2. 데이터 디렉터리 내에 `backups` 폴더를 개설하고, `backup_YYYYMMDD_HHMMSS.zip` 이름의 압축 파일을 생성합니다.
  3. 실재하는 데이터 파일들(`transactions.jsonl`, `categories.jsonl`, `budgets.jsonl`, `recurring.jsonl`)을 순차 압축하여 안전히 잠근 뒤 파일 위치를 표시합니다.

### 2.13 [recurring] - 반복 고정비 자동 관리 메뉴
* **진입 조건**: `recurring` 입력
* **동작 및 흐름**:
  1. 대화식 하위 작업 선택지를 띄웁니다:
     ```text
     [ 반복 고정 거래 및 생성 ]
     1. 등록된 반복 템플릿 목록 조회
     2. 신규 반복 템플릿 등록
     3. 기존 반복 템플릿 삭제
     4. 특정 월 반복 거래 일괄 생성
     선택 (1/2/3/4/엔터(취소)): 
     ```
  2. 세부 분기:
     * **1번 (목록)**: `REC-XXXXXX` 번호와 매월 반복 일자(`매월 X일`), 카테고리, 금액, 메모, 태그들을 가지런히 표로 렌더링합니다.
     * **2번 (등록)**: `add`와 동일하게 타입, 카테고리, 금액을 받으며, `매월 반복 일자(1-31):` 및 메모, 태그를 추가로 받아 템플릿 데이터베이스에 등록합니다.
     * **3번 (삭제)**: `삭제할 템플릿 ID (REC-XXXXXX):` 대기 후 일치 건을 찾아 unregister 처리합니다.
     * **4번 (생성)**: `반복 내역을 생성할 대상 월 (YYYY-MM):` 입력을 받습니다. 해당 월의 일수(예: 2월 28일/29일)를 파악하고, 반복일이 마지막 날짜보다 큰 경우 해당 월의 말일로 조율하여 거래 내역으로 생성합니다. 이때 기존 트랜잭션들과의 중복 등록(동일 날짜, 카테고리, 금액, 메모, `recurring` 태그 확인)을 방지하는 정합성 스캔 후 누락분만 안전하게 가계부에 덧붙입니다.

### 2.14 [exit / quit] - 안전 종료
* **진입 조건**: `exit` 또는 `quit` 입력
* **동작 및 흐름**:
  1. 셸 무한루프(`while True`)를 안전하게 탈출(break) 시킵니다.
  2. 실행 종료 작별 인사를 정답게 출력합니다.
     ```text
     ==================================================
        이용해 주셔서 감사합니다. 가계부를 종료합니다! 
     ==================================================
     ```
  3. 가계부 프로세스를 안전하게 종료하고 부모 터미널 셸로 매끄럽게 제어권을 복귀시킵니다.
