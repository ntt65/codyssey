가계부나 로그 데이터를 저장할 때 가장 많이 고민하게 되는 두 포맷입니다.
결론부터 말씀드리면, JSON은 데이터 전체를 하나의 거대한 덩어리로 저장하는 방식이고, JSONL(JSON Lines)은 데이터를 한 줄에 하나씩 쪼개서 저장하는 방식입니다. [1] 
가계부 시스템을 예로 들어 두 방식의 차이점을 명확하게 비교해 드릴게요.
------------------------------
## 1. JSON (JavaScript Object Notation)
가장 대중적인 데이터 교환 포맷입니다. 전체 데이터를 하나의 커다란 리스트([])나 객체({})로 감싸서 표현합니다. [2] 

* 가계부 예시 (ledger.json)

[
  {"id": 1, "date": "2026-06-11", "category": "식비", "amount": 12000},
  {"id": 2, "date": "2026-06-12", "category": "교통비", "amount": 1500}
]

* 장점: 데이터 구조가 한눈에 직관적으로 보입니다. 대다수 프로그래밍 언어와 웹 환경에서 기본적으로 지원합니다.
* 단점: 가계부에 데이터가 1,000만 개가 있다고 가정해 봅시다. 마지막 줄에 내역을 딱 하나만 추가(Append)하려 해도, 파일 전체를 메모리에 다 읽어와서 해석(Parsing)한 뒤 다시 저장해야 합니다. 데이터가 커질수록 컴퓨터가 비명을 지르게 됩니다. [3, 4, 5] 

------------------------------
## 2. JSONL (JSON Lines / NDJSON)
새로운 대안으로 떠오른 포맷입니다. 전체를 감싸는 대괄호([])나 쉼표(,)가 없고, 완전한 JSON 객체 하나가 한 줄(\n)씩 차지하는 형태입니다.

* 가계부 예시 (ledger.jsonl)

{"id": 1, "date": "2026-06-11", "category": "식비", "amount": 12000}
{"id": 2, "date": "2026-06-12", "category": "교통비", "amount": 1500}

* 장점: 새로운 가계부 내역이 생기면, 기존 파일을 읽을 필요 없이 파일 맨 끝에 그냥 텍스트 한 줄만 슥 이어 붙이면(Append) 끝납니다. 속도가 압도적으로 빠르고 안전합니다.
* 단점: 파일 전체를 하나의 덩어리로 처리하는 프로그램(웹 브라우저 등)에 바로 집어넣으면 문법 오류가 납니다. 한 줄씩 읽어서 처리하는 코드를 따로 짜야 합니다.

## 💡 방금 배운 '제너레이터'와 JSONL의 환상적인 궁합!
앞서 배운 제너레이터 문법이 빛을 발하는 곳이 바로 이 JSONL 파일을 읽을 때입니다. 10GB짜리 거대한 가계부 파일이어도 메모리를 거의 쓰지 않고 한 줄씩 읽어 들일 수 있습니다.

# JSONL 파일을 제너레이터로 읽는 파이썬 코드 예시import json
def read_large_jsonl(file_path):
    with open(file_path, "r", encoding="utf-8") as f:
        for line in f:
            # 파일에서 한 줄씩 읽어와 즉시 JSON 객체(딕셔너리)로 변환 후 던져줌!
            yield json.loads(line)
# 대용량 파일이어도 메모리 단 1줄 분량만 쓰며 가계부 검색 가능for record in read_large_jsonl("ledger.jsonl"):
    if record["amount"] > 100000:
        print(f"🚨 고액 지출 발견: {record['category']} - {record['amount']}원")

## 1. 일반 텍스트 파일로 읽어서 추가할 때의 치명적인 문제점## ① 쉼표(,) 문법 오류 문제
JSON 리스트는 데이터 사이에 반드시 쉼표(,)가 있어야 합니다. 맨 마지막 괄호만 지우고 데이터를 넣으면 문법이 깨집니다.

* 기존 데이터: [ {"id":1, "amount":5000} ]
* 마지막 ]를 지우고 그냥 어펜드하면: [ {"id":1, "amount":5000} {"id":2, "amount":3000} ] ➡️ ❌ 쉼표가 없어서 JSON 파싱 에러 발생!
* 해결하려면: 괄호를 지운 자리에 ,를 먼저 쓰고 새로운 데이터를 넣어야 해서 코드가 아주 지저분해집니다.

## ② 하드디스크(File System)의 동작 방식 한계
컴퓨터는 파일의 '맨 끝에 내용을 덧붙이는 것(Append)'은 엄청나게 잘합니다. 하지만 '파일의 마지막 1바이트를 지우는 것(Truncate)'은 의외로 컴퓨터에게 꽤 복잡한 연산입니다. 파일 크기를 뒤에서부터 정확하게 깎아내야 하기 때문입니다.

## 2. 그럼에도 불구하고 억지로 구현한 파이썬 코드 예시
어떻게 작동하는지 원리를 이해하기 위해, 파이썬의 파일 탐색 기능(seek)을 이용해 파일의 맨 끝으로 이동한 뒤 마지막 괄호를 지우고 강제로 데이터를 끼워 넣는 코드를 보여드릴게요.

import os
def append_to_json_raw(file_path, new_data_string):
    # 1. 읽기+쓰기(r+) 모드로 파일을 텍스트 형식으로 엽니다.
    with open(file_path, "r+", encoding="utf-8") as f:
        
        # 2. 파일의 맨 끝(오프셋 0, 끝 기준 2)으로 커서를 이동시킵니다.
        f.seek(0, os.SEEK_END)
        pos = f.tell() # 현재 파일의 총 크기(바이트)
        
        # 3. 뒤에서부터 공백이나 닫는 괄호(']')를 찾을 때까지 거꾸로 탐색합니다.
        while pos > 0:
            pos -= 1
            f.seek(pos, os.SEEK_SET)
            char = f.read(1)
            
            if char == ']':
                # 4. 닫는 괄호를 찾았다면, 그 앞 위치로 파일 크기를 딱 잘라버립니다 (지우기 효과)
                f.seek(pos, os.SEEK_SET)
                f.truncate() 
                break
        
        # 5. 문법에 맞게 쉼표(,)를 붙이고 새 데이터와 닫는 괄호를 강제로 어펜드합니다.
        f.write(f", {new_data_string}]")
# 사용 예시# append_to_json_raw("ledger.json", '{"id": 3, "amount": 1500}')

## 3. 왜 이렇게 안 하고 JSONL을 쓸까요?
위 코드에서 보시다시피 겨우 가계부 데이터 1줄 추가하려고 파일의 바이트 단위를 거꾸로 뒤지며 하드디스크 커서를 움직이는 위험한 작업을 해야 합니다. 만약 이 와중에 컴퓨터 전원이 꺼지거나 에러가 나면 가계부 파일 전체가 깨져서 열리지 않게 됩니다.
반면 JSONL은 이런 꼼꼼한 계산이 전혀 필요 없습니다.

# JSONL의 깔끔하고 안전한 추가 방식with open("ledger.jsonl", "a", encoding="utf-8") as f:
    f.write('{"id": 3, "amount": 1500}\n') # 묻지도 따지지도 않고 맨 뒤에 한 줄 툭! 끝!

질문자님이 생각하신 "맨 뒤를 지우고 이어 붙이기" 아이디어는 개발자들의 오랜 고민과 정확히 일치합니다. 그리고 그 고민을 가장 우아하고 안전하게 해결하기 위해 대괄호[]와 쉼표,를 아예 없애버린 결과물이 바로 JSONL 포맷인 것입니다.
컴퓨터가 파일을 다루는 내부 매커니즘까지 파고드는 아주 훌륭한 접근이었습니다!
