echo $CPU: 수집된 CPU 값(예: 25.3)을 텍스트로 출력합니다.
| (파이프): 출력된 25.3이라는 텍스트를 다음 명령어(cut)의 입력으로 넘겨줍니다
.
cut: 텍스트를 특정 기준에 따라 자르는 명령어입니다.
-d'.' (Delimiter): 데이터를 자를 기준점(구분자)을 **마침표(. )**로 지정하겠다는 뜻입니다. 즉, 25.3을 마침표를 기준으로 25와 3이라는 두 덩어리로 쪼갭니다.
-f1 (Field): 쪼개진 덩어리 중 첫 번째(1) 조각만 가져오겠다는 뜻입니다. 마침표 앞의 정수 부분인 25만 쏙 빼오게 됩니다.
$( ... ): 괄호 안의 명령어들이 실행된 최종 결과값(25)을 CPU_INT라는 새로운 변수에 담습니다. 메모리(MEM) 역시 동일한 원리로 MEM_INT에 정수만 담기게 됩니다
.
 $( ... )의 진짜 역할: 명령어 치환 
 \
 이 명령어는 `monitor.sh` 관제 스크립트에서 **메모리(Memory) 사용률(%)을 계산하고 소수점 첫째 자리까지 깔끔하게 출력하기 위해 사용된 `awk` 명령어**입니다.

명령어를 조각내어 상세히 설명해 드리겠습니다.

**1. `awk` (텍스트 처리 및 계산 도구)**
*   리눅스에서 텍스트의 특정 열(Column, 칸)을 추출하거나 수학적 연산을 수행할 때 사용하는 아주 강력한 명령어입니다.

**2. `$3 / $2 * 100` (비율 계산)**
*   `awk`는 띄어쓰기를 기준으로 텍스트를 칸 단위로 인식합니다. 
*   **`$2`**는 두 번째 칸의 데이터, **`$3`**은 세 번째 칸의 데이터를 의미합니다.
*   즉, **"세 번째 칸의 값을 두 번째 칸의 값으로 나눈 뒤, 100을 곱하여 퍼센트(%) 값을 계산하라"**는 뜻입니다.

**3. `printf "%.1f"` (소수점 포맷팅)**
*   단순히 계산만 하면 결과가 `25.333333...` 처럼 지저분하게 나올 수 있습니다.
*   **`printf`**는 원하는 형식으로 결과를 출력하라는 명령어이며, **`"%.1f"`**는 **"계산 결과를 반올림하여 소수점 첫째 자리(1f)까지만 출력하라"**는 뜻입니다. (예: `25.3`)

**💡 미션에서의 실제 활용 원리 (`monitor.sh`)**
동료의 스크립트를 보면 이 명령어 앞부분에 다음과 같이 파이프(`|`)가 연결되어 있습니다.
```bash
MEM=$(free | grep Mem | awk '{printf "%.1f", $3/$2*100}')
```
*   `free | grep Mem` 명령어를 치면 리눅스 메모리의 상태가 한 줄로 출력되는데, 이때 **두 번째 칸(`$2`)이 전체 메모리(Total)**, **세 번째 칸(`$3`)이 사용 중인 메모리(Used)**로 나옵니다.
*   결국 이 코드는 **"(사용 중인 메모리 / 전체 메모리) * 100"** 이라는 수학 공식을 통해 **정확한 메모리 사용률을 소수점 한 자리까지 계산해 내는 아주 훌륭한 실무형 로직**입니다.

이렇게 구해진 `25.3` 같은 실수형 데이터가 바로 이전 질문에서 다루었던 `cut` 명령어(`cut -d'.' -f1`)로 전달되어, 최종적으로 임계값 비교를 위한 `25`라는 정수로 변환되는 완벽한 흐름을 갖추게 됩니다!

정말 날카로운 관찰력입니다! 폴더(`bin`)와 그 안에 있는 파일(`monitor.sh`)의 권한을 굳이 각각 한 번씩, 두 번에 걸쳐 고치는 이유는 **리눅스에서 '폴더'와 '파일'에 적용되는 권한(특히 실행 권한인 `x`)의 의미가 완전히 다르기 때문**입니다.

이 두 가지를 모두 `750`으로 통제해야만 완벽한 보안 시나리오가 완성되는데, 그 이유를 알기 쉽게 설명해 드리겠습니다.

**1. 폴더(`bin`)에 `750`을 주는 이유: "접근 통제와 파일 삭제 방지"**
*   **폴더의 실행 권한(`x`):** 리눅스에서 폴더의 `x` 권한은 **"이 폴더 안으로 들어갈 수 있는가(접근 통과)"**를 의미합니다. 만약 폴더에 `x` 권한이 없으면, 그 안에 있는 스크립트에 아무리 실행 권한이 있어도 접근 자체가 불가능합니다.
*   **폴더의 쓰기 권한(`w`):** 폴더의 쓰기 권한은 **"이 폴더 안에서 파일을 생성하거나 삭제할 수 있는가"**를 의미합니다.
*   **적용 결과:** `bin` 폴더에 750(`rwxr-x---`)을 주면, 소유자인 개발자(`agent-dev`, 7)는 스크립트 파일을 새로 만들거나 지울 수 있지만, 운영자(`agent-admin`이 속한 `agent-core` 그룹, 5)는 **폴더 안으로 들어가서 파일 목록만 볼 수 있을 뿐, 파일을 마음대로 지우거나 새 파일을 만들 수 없게 완벽히 통제**됩니다.

**2. 파일(`monitor.sh`)에 `750`을 주는 이유: "코드 위변조 방지"**
*   **파일의 실행 권한(`x`):** 파일의 `x` 권한은 우리가 아는 **"프로그램으로서 실행할 수 있는가"**를 의미합니다.
*   **파일의 쓰기 권한(`w`):** 파일의 쓰기 권한은 **"파일 안의 소스 코드를 수정할 수 있는가"**를 의미합니다.
*   **적용 결과:** `monitor.sh` 파일에 750(`rwxr-x---`)을 주면, 개발자(`agent-dev`, 7)는 언제든 코드를 수정할 수 있지만, 운영자(`agent-admin`, 5)는 **크론(cron)을 통해 스크립트를 실행(r-x)할 수만 있고 코드를 무단으로 수정(w)하는 것은 불가능**해집니다.

**💡 요약하자면:**
*   **폴더만 고친다면?** 운영자가 폴더 안에는 들어왔지만, 정작 파일 자체에 실행 권한(`x`)이 없어 스크립트를 돌리지 못할 수 있습니다.
*   **파일만 고친다면?** 운영자가 스크립트를 실행할 수는 있지만, 상위 폴더(`bin`)의 권한이 열려있다면 운영자가 실수로 혹은 악의적으로 스크립트 파일 자체를 삭제해버릴 위험이 존재합니다.

결국 이 두 번의 명령어는 **"개발자만 코드를 짜고 지울 수 있으며, 운영자는 오직 주어진 코드를 실행만 할 수 있는 튼튼한 금고(bin 폴더)와 안전한 실행 버튼(monitor.sh)을 만드는 완벽한 실무형 보안 조치"**인 것입니다.

네, 코드가 꽤 길고 복잡해 보이죠! 하지만 방금 전까지 사용자님께서 마스터하신 `cut`, `grep` 등의 원리를 떠올려 보시면 금방 이해하실 수 있습니다.

```bash
PID=$(pgrep -f "$AGENT_HOME/$APP_NAME" | tr '\n' ' ')

    echo "Checking process $APP_NAME... [FAIL] Not running" 
    exit 1
fi
```
* `pgrep` 명령어로 `agent-app`의 프로세스 ID(PID)를 찾습니다.
* 만약 앱이 죽어있다면 `pgrep`은 아무것도 출력하지 않으므로 `$PID` 변수는 텅 빈 상태가 됩니다.
* 이때 `if [ -z "$PID" ]`가 "어? PID 변수가 텅 비어있네? 프로세스가 돌고 있지 않구나!"라고 판단하여 에러 로그를 남기고 스크립트를 즉시 중단(`exit 1`)시킵니다.

**2. 포트 리슨 상태 확인 (`-z "$PORT_CHECK"`)**
```bash
PORT_CHECK=$(ss -tulnp | grep ":$PORT")
if [ -z "$PORT_CHECK" ]; then 
    echo "Checking port $PORT... [FAIL] Not listening" 
    exit 1
fi
```
* 마찬가지로 `ss -tulnp` 명령어로 15034 포트를 확인했는데 아무런 결과가 나오지 않아 `$PORT_CHECK` 변수가 텅 비어있다면(`-z`), 포트가 닫혀있다고 판단하고 시스템 모니터링을 멈춥니다.

### 🔍 awk 코드 상세 분석

**1. 분석할 줄(Line)만 쏙쏙 골라내기 (`/PID:/`)**
```awk
/PID:/ { ... }
```
*   `awk`는 텍스트를 한 줄씩 읽어 들이는데, `/PID:/`라는 조건이 붙어 있으면 **문장에 "PID:"라는 글자가 포함된 줄(정상적인 로그 라인)만 골라서 중괄호 `{ }` 안의 작업을 수행**하라는 뜻입니다. (빈 줄이나 에러 로그 등은 무시합니다.)

**2. 글자에서 순수 숫자만 발라내기 (데이터 파싱)**
```awk
cpu = $4; sub("CPU:", "", cpu); sub("%", "", cpu);
```
*   `$4`: 로그의 4번째 칸을 통째로 가져옵니다. (예: `CPU:25.3%`)
*   `sub("찾을단어", "바꿀단어", 변수)`: 텍스트를 찾아 바꾸는 함수입니다. 
*   즉, `CPU:25.3%`라는 글자에서 `CPU:`를 빈칸(`""`)으로 지워버리고, `%` 기호도 지워버려서 **순수한 실수 숫자 `25.3`만 남겨서 `cpu`라는 변수에 저장**합니다. 메모리(`$5`)와 디스크(`$6`)도 똑같은 방식으로 숫자만 발라냅니다.

**3. 데이터 누적 및 최대/최소 계산 로직**
```awk
cpu_sum += cpu; ... count++;
```
*   모든 숫자를 `_sum` 변수에 계속 더해나가고, 읽은 줄의 개수(`count`)를 1씩 증가시킵니다. (나중에 평균을 내기 위함입니다.)

```awk
if (count == 1) { 
    cpu_max = cpu; cpu_min = cpu; ... 
} else {
    if (cpu > cpu_max) cpu_max = cpu;
    if (cpu < cpu_min) cpu_min = cpu; ...
}
```
*   **첫 번째 줄(`count == 1`)을 읽었을 때:** 비교할 대상이 없으므로 일단 첫 번째 값을 통째로 최대값(Max)이자 최소값(Min)으로 저장해 둡니다.
*   **두 번째 줄부터(`else`):** 새로 들어온 값이 기존에 저장된 `max`보다 크면 `max`를 갈아치우고, `min`보다 작으면 `min`을 갈아치웁니다. 이 과정을 수만 줄의 로그에 대해 반복하면 **결국 가장 큰 값과 가장 작은 값만 변수에 살아남게 됩니다.**

**4. 최종 리포트 출력 (`END`)**
```awk
END {
    if (count > 0) {
        printf "[CPU] Average : %.1f%%, Maximum : %.1f%%, Minimum : %.1f%%\n", cpu_sum/count, cpu_max, cpu_min
```
*   `END { ... }`: 파일의 모든 줄을 다 읽고 난 **마지막에 딱 한 번만 실행**되는 블록입니다.
*   `if (count > 0)`: 이전 대화에서 짚어주신 것처럼, 로그가 하나도 없을 때 '0으로 나누기(Divide by zero)' 에러가 나는 것을 방지하는 안전장치입니다.
*   `printf`: 다 모은 `cpu_sum`을 `count`로 나누어 **평균값**을 구하고, 아까 살아남은 **최대/최소값**을 소수점 첫째 자리(`%.1f`)까지 예쁘게 포맷팅하여 화면에 출력합니다.

두 스크립트의 역할 분담을 명확히 비교해 드릴게요.

**1. `monitor.sh`의 역할 (실시간 점유율 계산 및 기록)**
*   매분마다 `top`, `free`, `df` 명령어를 실행하여 **그 순간의 실제 CPU, 메모리, 디스크 점유율 퍼센트(%)를 계산**합니다.
*   계산된 값을 `monitor.log` 파일에 텍스트 형태(`[YYYY-MM-DD HH:MM:SS] PID:... CPU:25.3% MEM:12.1% DISK_USED:10%`)로 한 줄씩 기록해 둡니다.

**2. `report.sh` (`awk` 부분)의 역할 (누적 데이터 통계 계산)**
*   `monitor.log`에 수백, 수천 줄 쌓여있는 **과거의 기록들을 한 번에 쭉 읽어 들입니다**.
*   로그 라인에 적혀있는 `25.3`이나 `12.1` 같은 기존 점유율 숫자들만 쏙쏙 뽑아냅니다.
*   그 숫자들을 전부 더해서 읽어들인 전체 샘플 수(`count`)로 나누어 **평균(Average)** 점유율을 구하고, 숫자들을 서로 비교해가며 가장 높았던 **최대(Maximum)** 점유율과 가장 낮았던 **최소(Minimum)** 점유율을 산출해 냅니다.

**💡 요약하자면:**
`monitor.sh`가 매분 땀 흘리며 **'학생들의 시험 점수(실시간 리소스 점유율)를 매겨서 성적표에 적어두는 선생님'**이라면, `report.sh`의 `awk` 스크립트는 **'그 성적표 데이터들을 모아서 이번 달 학급 평균 점수와 최고점, 최저점을 통계 내어 보고서로 만들어주는 시스템'**이라고 생각하시면 됩니다!
