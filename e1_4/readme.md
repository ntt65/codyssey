## 1. 문제 정의

  온라인 이커머스 환경에서는 상품을 장바구니에 담고도 실제 구매로 이어지지 않는 경우가 매우 빈번하게 발생하며, 이는 전환율 저하의 가장 핵심적인 원인으로 작용한다. 특히 구매 직전 단계에서의 망설임과 의사결정 지연은 사용자 이탈로 직접적으로 이어지며, 이는 정보 이해의 어려움과 판단 불확실성에서 비롯되는 문제로 볼 수 있다.

- Baymard Institute(2023)에 따르면 장바구니 이탈률은 평균 약 70% 수준에 달하며, 배송비, 반품 정책, 옵션 정보 등에 대한 불확실성이 주요 이탈 원인으로 나타난다.
- Nielsen Norman Group의 연구에 따르면 온라인 사용자는 웹페이지를 꼼꼼히 읽기보다 빠르게 훑어보는 경향이 있으며, 필요한 정보를 빠르게 찾지 못할 경우 이탈 가능성이 높아진다. 또한 상세페이지에 정보가 존재하더라도 이를 쉽게 파악할 수 없는 구조에서는 구매로 이어지기 어렵다는 점이 확인되었다.
- Guo & Agichtein(2010)의 연구에서는 마우스 이동, 스크롤 패턴, 체류 시간과 같은 행동 데이터가 사용자 참여도 및 관심 수준과 유의미한 상관관계를 가지는 것으로 나타났으며, 이를 통해 사용자 행동 데이터를 기반으로 구매 의도와 망설임을 추정할 수 있음을 보여준다.

  이에 본 프로젝트는 사용자 행동 데이터를 기반으로 구매 과정에서의 망설임을 실시간으로 감지하고, 해당 시점에 적절한 개입을 통해 의사결정을 유도하는 시스템을 제안한다. 단순한 추천이나 분석을 넘어, 실제 구매 흐름 내에서 사용자의 행동에 직접적으로 영향을 미치는 실시간 개입형 서비스라는 점에서 가치가 있으며, 전환율 개선이라는 실질적인 문제 해결 측면에서도 의미를 가진다.

## 2. 타겟 사용자

### (1) 주요 사용자 정의

  본 서비스는 사용자 행동 데이터를 기반으로 구매 전환을 개선하는 솔루션으로, 온라인 쇼핑몰의 전환율 개선이 필요한 사업자(B2B)를 주요 사용자로 설정한다. 특히 별도의 개발 리소스가 부족하지만, 매출 성과 개선이 필요한 패션·뷰티 중심의 중소형 쇼핑몰 운영자 및 자사몰 사업자를 핵심 타겟으로 한다.

### (2) 사용자 니즈 분석

- 쇼핑몰 운영자: 구매 직전 이탈을 줄이고 전환율을 개선하고자 하며, 고객의 망설임 원인을 파악할 수 있는 행동 데이터와 이를 실제 성과로 연결할 수 있는 즉각적인 개선 방안을 필요로 한다. 또한 별도의 복잡한 개발 없이 빠르게 도입 가능한 솔루션을 선호한다.
- 최종 소비자(간접 사용자): 상품 선택 과정에서 필요한 정보를 빠르게 확인하고 싶어하며, 사이즈나 색상 등 선택에 대한 불확실성을 줄이고자 한다. 또한 유사 사용자 경험을 참고하여 보다 쉽게 판단하고, 복잡한 탐색 없이 직관적으로 구매 결정을 내리기를 원한다.

## 3. 기존 서비스와의 차별점

  사용자 행동 데이터를 기반으로 히트맵, 세션 분석, 저니 분석 등을 제공하는 대표적인 UX 분석 도구로 위븐(Woven)과 뷰저블(Beusable)이 있다. 이들 서비스는 사용자 행동을 분석하는 데 초점을 두는 반면, 본 서비스는 실시간 개입을 통해 의사결정에 직접 영향을 미친다.

| 위븐(Woven), 뷰저블(Beusable) | concho |
| --- | --- |
|   • 사용자 행동 데이터를 기반으로 히트맵, 세션 분석, 저니 분석 기능 제공
  • 사용자 이탈 구간 및 UX 문제 분석
  • AI 기반 인사이트 리포트를 통한 데이터 해석 지원 |   • 사용자 행동을 실시간으로 해석하고 구매 과정에 직접 개입
  • 이탈 가능성이 높은 시점에 맞춰 즉각적인 정보 제공
  • 분석 결과를 리포트가 아닌 사용자 경험에 직접 반영
  • 별도의 후속 작업 없이 전환율 개선으로 연결되는 구조 |

## 4. 서비스 개요

### (1) 사용자 행동 기반 상태 분석 및 이탈 감지 기능

  웹사이트에 삽입된 FullStory 스크립트를 통해 방문자의 마우스 움직임, 클릭 지연, 스크롤 패턴 등의 마이크로 인터랙션 데이터를 실시간으로 수집하고, 이를 분석하여 사용자의 구매 단계와 이탈 가능성을 판단하는 기능이다. 수집된 행동 데이터를 바탕으로 사용자의 현재 상태와 망설임 수준을 구조적으로 파악한다.

- 마우스 이동 경로 및 호버 시간 → 관심 영역 집중도 → 상품 요소 관심 상태
- 클릭 전 지연 시간 → 의사결정 지연 정도 → 구매 망설임 상태
- 스크롤 속도 변화 → 탐색 속도 및 패턴 → 정보 탐색 / 스캐닝 상태
- 페이지 체류 시간 → 전체 관심 수준 → 구매 의도 강도
- 페이지 이동 흐름 → 탐색 경로 → 탐색 / 비교 단계
- 빠른 스크롤 및 반복 탐색 → 정보 확인 후 불확실성 → 이탈 위험 상태

### (2) 실시간 맞춤 개입 기능

  사용자의 행동 분석 결과를 기반으로, 현재 상태와 망설임 원인에 맞는 정보를 실시간으로 제공하여 구매 의사결정을 유도하는 기능이다. 분석된 상태에 따라 개입 타이밍과 제공 콘텐츠를 동적으로 결정하여, 필요한 순간에 필요한 정보만 전달한다.

- 사이즈 / 옵션 반복 탐색 → 사이즈 관련 정보 제공 필요 → 사이즈 가이드 / 유사 체형 리뷰 제공
- 색상 이미지 집중 탐색 → 시각적 비교 정보 필요 → 색상 비교 이미지 / 실제 착용 후기 제공
- 배송 정보 영역 체류 → 배송 조건 확인 필요 → 예상 배송일 / 배송 정책 안내 제공
- 가격 확인 후 행동 정체 → 가격 관련 판단 지원 필요 → 할인 / 쿠폰 / 프로모션 정보 제공
- 장시간 탐색 후 빠른 스크롤 → 핵심 정보 재정리 필요 → 가격·옵션·배송·환불 조건 요약 제공

## 5. AI 기술 적용

### (1) FullStory JavaScript SDK

: 유저의 스크롤, 클릭, 체류 시간 등의 행동 데이터를 실시간으로 수집하고 분석하여 사용자 행동 패턴을 구분한다.

- 유저의 브라우저 상 대부분의 행동을 자동으로 캡처할 수 있어 별도의 수집 로직을 직접 구현할 필요가 없다.
- 임의로 트리거 기준을 설정하는 방식보다 데이터 기반으로 행동을 포착할 수 있어 초기 설정 부담이 적다.
- 특정 행동 이벤트를 자동으로 감지할 수 있어 망설임과 같은 미세한 행동을 놓치지 않고 분석할 수 있다.
- 수집된 데이터를 페이지 영역과 매핑하여 사용자 행동을 구조적으로 해석할 수 있다.

### (2) LLM (GPT-5 nano)

: 사용자 의도와 RAG 기반 컨텍스트를 입력받아, 적절한 액션 유형과 메시지를 생성한다.

- 창작이 아닌 분류와 단문 생성에 적합하여 본 서비스의 요구사항에 부합한다.
- 사용자 의도와 컨텍스트를 결합해 필요한 정보를 빠르게 선택하고 전달할 수 있다.
- 실시간 개입 서비스에 적합하도록 빠른 응답 속도를 제공한다.
- 비용 효율성이 높아 대량 요청 처리에 유리하다.

### (3) Vector DB (Pinecone)

: 사이즈·색상 리뷰, 쿠폰 정책, 행동 지침 등의 컨텍스트 데이터를 임베딩 형태로 저장하고, 사용자 의도에 맞는 유사 데이터를 검색하여 제공한다.

- LLM 단독 사용 시 발생할 수 있는 부정확한 응답(환각)을 방지할 수 있다.
- 임베딩 기반 검색을 통해 단순 키워드 매칭보다 정확한 컨텍스트 탐색이 가능하다.
- 관리형 서비스로 제공되어 별도의 인프라 구축 없이 빠르게 적용할 수 있다.
- 데이터 변경 시 모델 재학습 없이 DB만 갱신하면 되어 운영 효율성이 높다.
- 초기 단계에서 무료 요금제를 활용해 부담 없이 테스트할 수 있다.

## 7.  기대 효과

### (1) 비즈니스 관점: 전환율 향상 및 이탈 감소

- 실시간 데이터 분석을 통해 장바구니 이탈을 줄이고 구매 전환율을 향상시킬 수 있다.
- UI 개편 없이 기존 서비스 구조를 유지하면서 성과 개선이 가능하다.
- 사용자 행동 데이터를 확보하여 개입 전략을 지속적으로 개선할 수 있다.
- 사용자별 최적의 혜택을 제공함으로써 마케팅 비용 효율을 높일 수 있다.

### (2) 최종 사용자 관점: 의사결정 효율 및 경험 개선

- 실시간 맞춤형 정보 제공을 통해 불필요한 탐색 과정을 줄일 수 있다.
- 핵심 정보 중심 전달로 제품에 대한 이해도를 높일 수 있다.
- 반복적인 정보 탐색을 줄여 이용 과정에서의 피로도를 완화할 수 있다.
- 구매에 대한 확신을 높여 전반적인 서비스 이용 경험을 개선할 수 있다.

## 8. 향후 발전 방향

  현재 서비스는 외부 도구를 활용하여 사용자 행동 이벤트를 수집하고, 이를 기반으로 사용자 의도를 추정하는 방식으로 동작한다. 그러나 이러한 방식은 간접적인 패턴 해석에 의존하기 때문에, 특정 상황에서의 사용자 의도를 보다 정교하게 구분하는 데에는 한계가 있다.

  이에 향후에는 서비스 운영을 통해 축적된 행동 데이터를 기반으로 자체 행동 분석 모델을 구축하는 것을 목표로 한다. 이를 통해 사용자가 사이즈, 색상, 가격 등 어떤 요소에서 망설이고 있는지를 보다 직접적으로 판단할 수 있으며, 개입의 정확도를 높일 수 있다.

  또한 자체 모델을 활용하게 되면 외부 도구에 대한 의존도를 낮추고, 서비스의 핵심 기능을 더욱 고도화할 수 있다. 나아가 이러한 행동 분석 모델은 시장에서 범용적으로 제공되지 않는 영역이기 때문에, 향후에는 모델 자체를 독립적인 솔루션으로 확장할 가능성도 있다.
